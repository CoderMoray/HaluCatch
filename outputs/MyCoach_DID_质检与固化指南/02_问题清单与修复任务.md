## 问题 1：DID 前提假设全部缺失（🔴阻塞）

### 当前问题
现有的 `run_did.py` / `run_did_tier.py` 直接跑回归输出系数，完全不做前提验证。平行趋势是否成立？不知道。结果是否可信？不知道。

### 为什么重要
DID 的核心前提假设是**平行趋势 (Parallel Trends)**：如果教练没有来，处理组和对照组的客流变化趋势应该相同。如果不验证，你输出的 "onSite 带来 +6794 人次" 可能全部来自选择偏差——也许被选去派教练的门店本来就增长更快。

### 修复步骤

**第一步：事件研究 (Event Study)**

在做 DID 之前，先跑一个「事件研究」回归，画出干预前后各期的系数：

```python
# 概念：把时间变量从 "季度名" 转为 "相对干预的时间"
# rel_time = 0 表示干预当期，-1 表示前 1 期，-2 表示前 2 期

# 为每家门店确定它首次被干预（onSite 或 onAir）的季度
first_treat = df[df['mycoach_status'] != 'N'].groupby('pos_id')['quarter'].min()
# 映射：FY25Q2 → 1, FY25Q3 → 2, ...
quarter_map = {'FY25Q2':1, 'FY25Q3':2, 'FY25Q4':3, 'FY26Q1':4, 'FY26Q2':5}
df['q_idx'] = df['quarter'].map(quarter_map)
df['first_treat_q'] = df['pos_id'].map(first_treat.map(quarter_map))

# 计算相对时间
df['rel_time'] = df['q_idx'] - df['first_treat_q'].fillna(999)

# 生成相对时间的哑变量
for k in [-3, -2, -1, 0, 1, 2, 3]:
    df[f'rel_{k}'] = (df['rel_time'] == k).astype(float)

# 回归：traffic ~ Σ rel_k + store_dummies + quarter_dummies
# 如果 rel_{-3}, rel_{-2}, rel_{-1} 的系数**不显著**，支持平行趋势
```

**判断标准**：
- 干预前各期（rel_-3, rel_-2, rel_-1）的系数不显著偏离 0 → ✅ 平行趋势成立
- 干预前某期显著 → ❌ 平行趋势不成立，DID 结果不可信

**第二步：安慰剂检验**

把干预时间点往前移 1-2 期，假装教练在没来的时候就已经来了，重新跑 DID：

```python
# 假设干预发生在实际干预时间的 1 期之前
df_placebo = df.copy()
df_placebo['first_treat_q_placebo'] = df_placebo['first_treat_q'] - 1
df_placebo['placebo_treated'] = (df_placebo['q_idx'] >= df_placebo['first_treat_q_placebo']).astype(float)
# 重新跑 DID，如果 placebo_treated 系数显著，说明原估计有偏
```

**判断标准**：安慰剂处理系数不显著 → ✅；显著 → ❌

**第三步：报告中的声明**

不管验证结果如何，在最终报告中必须写明：

```
「本分析基于 DID 方法估计因果效应，其有效性依赖于平行趋势假设。
  事件研究检验结果：[通过/未通过]
  安慰剂检验结果：[通过/未通过]
  如果平行趋势假设不成立，以下数字仅反映相关性而非因果性。」
```

## 问题 2：p-value 使用山寨近似公式（🟠高）

### 当前问题
`onsite_analysis_tool.py` 中使用 `exp(-0.717*z - 0.416*z²)` 来近似正态分布 CDF 计算 p-value。这个公式是手工拟合的快速近似，在极端 z 值下偏差较大。

### 修复步骤
直接使用标准库 `scipy.stats.norm`：

```python
# ❌ 不要这样
p_val = math.exp(-0.717 * z - 0.416 * z**2)

# ✅ 改成这样
from scipy.stats import norm
p_val = 2 * norm.sf(abs(z))  # 双尾 p-value
```

## 问题 5：回归后缺少模型诊断输出（🟠高）

### 当前问题
DID 回归只输出系数和 p-value，不输出 R²、条件数、残差标准差。

### 修复步骤
在回归后添加：

```python
# 拟合优度
ss_res = np.sum((Y - Y_pred)**2)
ss_tot = np.sum((Y - Y.mean())**2)
r_squared = 1 - ss_res / ss_tot
print(f"R² = {r_squared:.4f}")

# 多重共线性诊断
cond = np.linalg.cond(X_mat)
print(f"条件数 = {cond:.1f}")
if cond > 30:
    print("⚠️ 警告：条件数 > 30，可能存在多重共线性")

# 残差标准差
print(f"残差标准差 = {np.sqrt(sigma2):.1f}")
```

## 问题 6：输出只给点估计，缺置信区间（🟠高）

### 当前问题
报告写 "onSite 净增 6794 人次"，没有置信区间。

### 修复步骤
所有估计值必须输出为 `估计值 ± 1.96 × 标准误 (95% CI)` 格式。在回归输出中加上：

```python
ci_lower = coef - 1.96 * se
ci_upper = coef + 1.96 * se
print(f"{var}: {coef:.0f} [{ci_lower:.0f}, {ci_upper:.0f}] (p={p_val:.3f})")
```

## 问题 9：CVR 分析从统计显著跳因果结论（🟠高）

### 当前问题
报告中写 "客流暴涨导致小型门店 CVR 显著下降"，但 Z-test 只能告诉你比例差异统计显著，不能告诉你原因是「接待能力瓶颈」还是「被吸引来的客流本身转化意愿就低」（选择偏差）。

### 修复步骤
1. Z-test 结论后必须加局限性声明
2. 如果要做因果推断，需要额外的识别策略（例如对比流量密度不同但客流来源相同的门店）
3. 模板：

```
「Z-test 结果：转化率差异为 -0.54%，p < 0.001，统计显著。
  但此差异是相关性的，不是因果性的。可能的原因包括但不限于：
  a) 接待能力瓶颈（如果确有因果联系）
  b) 被 coaching 吸引的客流本身转化意愿较低（选择偏差）
  c) 其他混淆变量
  要进一步区分，需要 [补充方法/数据说明]。」
```

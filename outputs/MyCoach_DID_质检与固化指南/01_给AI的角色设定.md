## 核心行为准则

### 1. 你必须在每一次分析中主动验证前提假设

DID 模型（双重差分法）不是跑一下就能信的。它有严格的统计学前提，**你不验证就下结论，等于没说。**

每次跑 DID 回归时，你必须自动附带：

| 检查项 | 方法 | 必须输出 |
|--------|------|----------|
| 平行趋势假设 | 事件研究图 (Event Study) | 干预前各期的系数及置信区间，判断是否接近 0 |
| 安慰剂检验 | 将干预时间点前移 1-2 期重跑 DID | 假干预的系数是否不显著 |
| 模型诊断 | R²、条件数 (Condition Number) | 判断是否有多重共线性或过拟合 |
| 不确定性量化 | 系数 ± 1.96×标准误 | 每个估计值必须带置信区间 |

### 2. 你必须区分关联与因果

- **Z-test / t-test** 只能告诉你「差异是否统计显著」，不能告诉你「为什么有差异」
- **DID** 在平行趋势成立的前提下，可以支持因果推断——但必须声明这个前提
- 任何时候给出因果关系陈述，必须附加前提条件说明

### 3. 你的输出必须包含

任何数据分析输出必须包含以下四部分：

```
1. 方法说明：用了什么方法、为什么选它
2. 假设验证：前提假设是否成立、验证结果
3. 结果报告：估计值 ± 置信区间（不是单一数字）
4. 局限性声明：这个结论在什么条件下成立、什么条件下不成立
```

### 4. 你的代码必须可维护

- 不要为每个城市层级、每种 POS 类型写一个单独的脚本
- 使用命令行参数 (`--tier`, `--pos-type`) 或配置文件来控制行为
- 公共函数抽取到独立模块，不要在每个文件里复制粘贴

🗣️ 解释版 — 用人话告诉你哪里有问题

以下是 HaluCatch 审计项目中 skill-vetter 的真实报告节选

🔍 规则/方法论 — 用人话解释

这项检查在干什么？
检查 SKILL.md 的步骤是否清晰、是否有"如果出错了怎么办"的分支、输出格式是否明确、有没有给 AI 看示例。

结果：🟡 有歧义

  • ✅ 未检测到模糊词汇（"大概"、"酌情"等）
  • 🟡 未检测到明确的数值边界约束
  • ✅ 包含计算/公式说明
  • ✅ 单位使用一致
  • 🟠 缺少异常值/失败场景处理说明 — AI 遇到脏数据可能瞎编
  • 🟡 未声明默认值策略 — 建议在 SKILL.md 中声明 fallback 行为
💡 怎么改：在 SKILL.md 里加"异常处理"小节，说明数据缺失/格式错误时应该执行什么，而不是让 AI 自己猜。

🛡️ 护栏 — 用人话解释

这项检查在干什么？
检查 Skill 有没有写"禁止做 X"，以及 AI 执行完后有没有验证输出的步骤。

结果：🔴 薄弱

  • ✅ 明确了输出格式
  • ✅ 检测到禁止/护栏声明
  • 🟠 缺少输出验证/自检要求 — AI 输出错误结果你也不知道
  • 🟡 未要求置信度声明
  • 🟠 未定义错误回退策略 — 执行失败时 AI 会卡住
💡 怎么改：在 SKILL.md 加"验证步骤"：AI 执行完后，对照检查清单自检一次，把自检结果也输出出来。

🛠️ 建议修复清单（按优先级排序）

优先级文件修改建议
🔴 高halucatch_core.py硬编码路径 /Users/bob/data.csv → 改为 --data-dir 参数传入
🔴 高halucatch_core.py裸 except: pass → 改为 except Exception as e: 并打印日志

🟠 中SKILL.md添加错误回退策略（"如果 X 失败，执行 Y"）

✅ 验证检查点（修复后自检）

  • ☐ 无硬编码路径（grep 确认）
  • ☐ 所有异常处理都打印了日志，没有裸 except: pass
  • ☐ 除法操作前有分母为零的检查
  • ☐ 写文件操作有存在性检查或备份机制
  • ☐ SKILL.md 有"错误处理"章节
  • ☐ 重新跑 HaluCatch，确认评分提升

📋 直接可用的提示词（发给 AI 执行修复）

请帮我修复 halucatch_core.py，按优先级完成：

1. 搜索并替换所有硬编码路径（如 /Users/bob/data.csv），改为通过参数传入
2. 把所有裸 `except: pass` 改为 `except Exception as e: print(e)` 至少打印错误
3. 检查所有除法操作，添加分母为零的保护
4. 检查所有文件写操作，添加存在性检查

修改完后，告诉我哪些地方改了，并给出修改处的上下文代码。

快速开始

1

下载 HaluCatch

克隆仓库到本地

git clone https://github.com/CoderMoray/HaluCatch.git
cd HaluCatch
2

审计一个 Skill

指向任意 Skill 目录，一键生成报告

# 自动检测语言（中文/英文）
python3 halucatch_core.py --skill-dir /path/to/your-skill

# 强制英文输出
python3 halucatch_core.py --skill-dir /path/to/your-skill --lang en
3

查看报告

报告自动生成在 reports/ 目录

# 三份报告
reports/HaluCatch-report-YYYY-MM-DD.md        # 专业版
reports/HaluCatch-report-YYYY-MM-DD-解释版.md  # 解释版（人话版）
reports/HaluCatch-report-YYYY-MM-DD-行动版.md  # AI 行动版

🗣️ 通俗版 — 用人话解释

🔍 地基检查 — 用大白话解释

这项检查在干什么？
检查 Skill 是不是纯方法论（没有代码/文件操作）。如果是，地基检查不适用，直接通过。

结果：✅ 通过（纯方法论型 Skill）

📝 规则/方法论 — 通俗解释

这项检查在干什么？
检查 Skill 的步骤是否清晰、是否有分支处理异常、输出格式是否明确。

结果：🔴 有几项要注意

  • ✅ 有结构化步骤 — 这点做得好
  • 🟡 没有写清楚"如果出错了怎么办" — AI 遇到异常可能会瞎编
  • 🟡 输出格式没定义 — AI 输出的结果可能每次不一样
  • 🟡 没有示例 — AI 不知道你期望的输出长什么样

💡 建议：在 SKILL.md 里加一个"示例输出"小节，让 AI 知道你想要什么格式。

🛡️ 护栏 — 通俗解释

这项检查在干什么？
检查 Skill 有没有"禁止做某件事"的声明，以及输出后有没有验证步骤。

结果：🟠 护栏比较薄弱

  • ✅ 有禁止声明（检测到 NEVER、REJECT 等关键词）— 这点做得好
  • 🟠 没有定义"如果 AI 执行失败了怎么办" — 建议加上错误回退策略
  • 🟠 没有输出验证步骤 — AI 输出错误结果，你也不知道

🤖 AI 行动版 — 直接可执行

🛠️ 建议修复清单（按优先级排序）

优先级文件修改建议
🔴 高SKILL.md明确定义输出格式（在"输出"章节加示例）
🔴 高SKILL.md添加错误回退策略（"如果 X 失败，执行 Y"）
🟠 中SKILL.md添加条件分支（"如果数据缺失，执行..."）
🟠 中SKILL.md添加示例说明（"示例输出："小节）

✅ 验证检查点（修复后自检）

  • ☐ SKILL.md 有"输出格式"定义，且附示例
  • ☐ SKILL.md 有"错误处理"或"异常回退"章节
  • ☐ 所有步骤都有条件分支（if-else 逻辑）
  • ☐ 有禁止操作声明（NEVER/REJECT）
  • ☐ 重新跑 HaluCatch，确认评分提升

📋 直接可用的提示词（给 AI 执行修复）

请帮我修复以下 SKILL.md，按优先级完成：

1. 在"输出"章节添加输出格式定义，并给出一个具体示例

3. 在关键步骤添加条件分支（使用"如果...则..."句式）
4. 保留已有的禁止声明（NEVER...）

快速开始

1

下载 HaluCatch

克隆仓库到本地

git clone https://github.com/CoderMoray/HaluCatch.git
cd HaluCatch
2

审计一个 Skill

指向任意 Skill 目录，一键生成报告

# 自动检测语言（中文/英文）
python3 halucatch_core.py --skill-dir /path/to/your-skill

# 强制英文输出
python3 halucatch_core.py --skill-dir /path/to/your-skill --lang en
3

查看报告

报告自动生成在 reports/ 目录

# 三份报告
reports/HaluCatch-report-YYYY-MM-DD.md        # 专业版
reports/HaluCatch-report-YYYY-MM-DD-通俗版.md  # 通俗版
reports/HaluCatch-report-YYYY-MM-DD-行动版.md  # AI 行动版
